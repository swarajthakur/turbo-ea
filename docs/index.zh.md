# Turbo EA 简介

### 什么是 Turbo EA？

**Turbo EA** 是一个现代化的、自托管的**企业架构管理**平台。它使组织能够在一个平台中记录、可视化和管理其业务与技术架构的所有组成部分。

### 本指南适用于谁？

本指南适用于**所有使用 Turbo EA 的人员** —— 企业架构师、IT 经理、业务分析师、开发人员和管理员。无论您是在评估平台、日常管理组织的 IT 架构，还是作为管理员配置系统，您都能在这里找到所需的信息。入门无需高级技术知识。

### 核心优势

- **全面可视化**：在一个平台中查看组织内所有的应用程序、流程、能力和技术。
- **明智决策**：可视化报告（投资组合、能力地图、依赖关系、生命周期、成本等）有助于评估技术基础设施的现状。
- **生命周期管理**：通过五个阶段跟踪每个技术组件的状态 —— 从规划到退役。
- **协作**：多用户可同时工作，支持可配置的角色、干系人分配、评论、待办事项和通知。
- **AI 驱动的描述**：一键生成卡片描述。Turbo EA 将网络搜索与本地或商业 LLM 结合，生成类型感知的摘要 —— 附带置信度评分和来源链接。在您的基础设施上完全运行以保护隐私，也可连接商业提供商（OpenAI、Google Gemini、Anthropic Claude 等）。完全由管理员控制：选择哪些卡片类型获取 AI 建议，选择搜索提供商和模型。
- **可视化图表**：使用嵌入式 DrawIO 编辑器创建架构图表，与卡片库存完全同步。
- **业务流程建模**：BPMN 2.0 流程编辑器，支持元素链接、审批工作流和成熟度评估。
- **ServiceNow 集成**：与 ServiceNow CMDB 双向同步，保持 EA 架构与 IT 运维数据的连接。
- **多语言**：支持英语、西班牙语、法语、德语、意大利语、葡萄牙语和中文。

### 核心概念

| 术语 | 含义 |
|------|------|
| **卡片** | 平台的基本元素。代表任何架构组件：应用程序、流程、业务能力等。 |
| **卡片类型** | 卡片所属的类别（应用程序、业务流程、组织等） |
| **关系** | 两张卡片之间的连接，描述它们的关联方式（例如「使用」、「依赖于」、「属于」） |
| **元模型** | 定义存在哪些卡片类型、它们有哪些字段以及它们如何相互关联的结构。完全由管理员可配置 |
| **生命周期** | 组件的时间阶段：规划、引入、活跃、淘汰、生命周期结束 |
| **清单** | 可搜索、可筛选的所有 card 列表（涵盖所有类型）。批量编辑、Excel/CSV 导入导出，以及可共享的已保存视图 |
| **报告** | 预置可视化：投资组合、能力地图、生命周期、依赖关系、成本、矩阵、数据质量和生命周期结束 |
| **BPM** | 业务流程管理 —— 使用 BPMN 2.0 编辑器对业务流程建模，将图表元素链接到 card，并评估成熟度、风险与自动化程度 |
| **PPM** | 项目组合管理 —— 将 Initiative card 作为完整项目管理：状态报告、Work Breakdown Structures、看板与甘特图、预算、成本，以及每个倡议的风险登记册 |
| **TurboLens** | AI 驱动的 EA 智能 —— 供应商分析、重复检测、现代化评估、5 步 Architecture AI 向导，以及合规扫描（EU AI Act / GDPR / NIS2 / DORA / SOC 2 / ISO 27001） |
| **EA Delivery** | 与 TOGAF 对齐的交付层 —— Statements of Architecture Work、Architecture Decision Records 以及 EA 全景级别的风险登记册 |
| **SoAW** | 架构工作说明书 —— TOGAF 中正式界定架构项目范围的文档 |
| **ADR** | 架构决策记录 —— 捕获决策的背景、备选方案与后果，含状态工作流和 card 链接 |
| **风险登记册** | EA 全景级别的 TOGAF 阶段 G 风险登记册，独立于 PPM 的倡议级风险。指派负责人会自动创建一项待办事项 |
| **网页门户** | 基于 slug 的公开只读视图，展示 EA 全景的一部分 —— 无需登录即可分享 |
| **MCP 服务器** | 通过 Model Context Protocol 提供的只读 AI 工具访问 —— 可从 Claude Desktop、Cursor、GitHub Copilot 等 MCP 客户端查询 EA 数据 |
| **RBAC** | 基于角色的访问控制 —— 应用级角色加上每张 card 的干系人角色，包含 50 多项细粒度权限 |
